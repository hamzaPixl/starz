## 1. Project Overview

**Elevator Pitch**: Starz is a personal tool that pulls all your GitHub starred repos, auto-categorizes them with AI, generates semantic embeddings, and provides a chat interface to instantly find any starred repo by asking natural language questions.

**Goals**:
1. Make GitHub stars searchable and browsable with AI-powered categorization
2. Enable natural language discovery ("find me a React animation library I starred")
3. Persist all star metadata locally for instant offline-capable search
4. Fully open-source, installable via a single command (`pip install starz` or `uvx starz`)

**Success Metrics**:
| Metric | Target | Baseline |
|--------|--------|----------|
| Sync time for 300 repos | < 3 minutes | N/A |
| Search relevance (top-5 hit rate) | > 80% | N/A |
| Total API cost per full sync | < $0.10 | N/A |
| Time to install and first sync | < 5 minutes | N/A |

**Target Users**:
- **Power developers**: Stars hundreds of repos, can never find them — needs instant semantic search
- **Tech leads/architects**: Uses stars as a "read later" bookmark — wants categorized reference library
- **Open-source enthusiasts**: Explores broadly — wants a personal "awesome list" auto-generated from stars

## 2. Domain Model

**Entities**:
| Entity | Description | Key Attributes |
|--------|-------------|----------------|
| Repo | A GitHub starred repository | full_name, description, language, topics, category, summary, readme_content |
| Embedding | Vector representation of a repo | repo_id, embedding (1536-dim float) |
| Category | AI-assigned classification | name (one of ~15 predefined labels) |
| SyncState | Current sync pipeline status | state, progress, last_synced_at |

**Relationships**:
- Repo → Embedding: 1:1 (each repo has exactly one embedding)
- Repo → Category: N:1 (many repos share a category)

**Bounded Contexts**:
- **Ingestion**: GitHub API fetching, README extraction, upsert — `services/github.py`
- **Intelligence**: Embedding generation, categorization, summarization — `services/embeddings.py`, `services/categorizer.py`
- **Discovery**: Vector search, keyword search, RAG chat — `services/search.py`, `services/chat.py`

**Domain Events**:
- `SyncStarted`: user triggers sync → pipeline begins fetching
- `ReposSynced`: all repos fetched and stored → triggers embedding + categorization
- `EmbeddingsGenerated`: embeddings stored → search becomes available
- `CategoriesAssigned`: AI labels applied → browsing by category enabled

## 3. Architecture Overview

**Components**:
- **Backend**: Python/FastAPI REST API serving all endpoints
- **Frontend**: Next.js SPA with shadcn/ui for search, browse, and chat
- **Database**: SQLite file + sqlite-vec extension (single file, zero infra)
- **External Services**: GitHub API (data), OpenAI API (embeddings), Anthropic API (categorization + chat)

**Integrations**:
| Service | Purpose | Direction |
|---------|---------|-----------|
| GitHub API | Fetch starred repos + READMEs | outbound |
| OpenAI API | Generate text-embedding-3-small vectors | outbound |
| Anthropic API (Haiku) | Batch categorization + summaries | outbound |
| Anthropic API (Sonnet) | RAG chat responses | outbound |

**Tech Stack**: Python 3.11+, FastAPI, SQLite, sqlite-vec, httpx, OpenAI SDK, Anthropic SDK, Next.js, React, Tailwind CSS, shadcn/ui

**Deployment**: Open-source PyPI package. `pip install starz` → `starz serve`. Pre-built Next.js frontend bundled as static files inside the Python package. Single port (7827) serves both API and web UI.

## 7. Risks & Assumptions

**Risks**:
| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|------------|
| sqlite-vec installation issues on macOS ARM | M | M | Fallback to pip wheel; arm64 confirmed |
| Large READMEs blow up embedding costs | L | M | Truncate to 4000 chars |
| GitHub rate limit during sync | L | M | 100/page pagination = ~4 requests; well under 5000/hr |
| Claude API rate limits on categorization | L | L | 17 batch calls; exponential backoff |
| Embedding model change breaks search | L | M | Store embedding_text for re-embedding |
| User has 1000+ stars (edge case) | M | L | Pipeline scales linearly, cost ~$0.05 |

**Assumptions**:
- User has `gh` CLI installed and authenticated (fallback: manual GITHUB_TOKEN in .env)
- User has OpenAI and Anthropic API keys
- READMEs are in English or embedding model handles multilingual adequately
- ~15 categories cover the diversity of typical star collections
- Requirements may change as user feedback is gathered
